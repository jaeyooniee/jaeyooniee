## <img src="https://media.giphy.com/media/QssGEmpkyEOhBCb7e1/giphy.gif" width="28"> Featured Projects

<details>
<summary><b>Systematic Trading Framework</b></summary>

<br/>

A modular Python framework for designing, comparing, and validating systematic trading strategies. Implements a target-weight architecture (`generate_target_weights()` → `rebalance()`) supporting Pairs Trading via cointegration, cross-sectional Momentum, Moving-Average Crossover, and RSI Mean Reversion, benchmarked against buy-and-hold.

| Attribute | Detail |
| :--- | :--- |
| **Stack** | Python · pandas · NumPy · statsmodels · Matplotlib |
| **Scale** | Multi-asset universe, vectorised portfolio backtesting |
| **Performance** | Sharpe, Sortino, Calmar, MDD, CAGR with 10,000-iteration bootstrap confidence intervals |
| **Security** | Deterministic, reproducible runs; isolated configuration; no live credentials in source |
| **Impact** | Unified comparison harness enabling consistent, statistically grounded strategy evaluation |
| **Repository** | [View Source](https://github.com/jaeyooniee/systematic-trading-backtesting-framework/tree/main) |

The framework was migrated from a discrete signal model (1 / 0 / −1) to a continuous target-weight allocation model, resolving state-versus-event signalling defects and a capital-allocation bug in trade execution. The result is a robust research platform that cleanly separates strategy logic, portfolio construction, and performance attribution.

</details>

<details>

<br/>

A full stack campus collaboration platform delivering authenticated, real-time one-to-one messaging. Built as a first-year team project and engineered with production patterns including authentication integration, content moderation, and live socket communication.

| Attribute | Detail |
| :--- | :--- |
| **Stack** | PHP · MySQL · Node.js · Socket.io · JavaScript |
| **Scale** | Real-time private chat rooms with persistent message storage |
| **Performance** | Low-latency bidirectional messaging via Socket.io private rooms |
| **Security** | CAS authentication integration; server-side profanity filtering; input validation |
| **Impact** | Delivered a working, demonstrable collaboration product as a coordinated team |
| **Repository** | [View Source](https://gitlab.cs.man.ac.uk/a70598hm/campus-connect) |

The application integrated centralised authentication, implemented a moderated messaging layer, and resolved cross-page UI consistency issues to ship a coherent end-to-end experience.

</details>
